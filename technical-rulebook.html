MODULE: WEBATHON

TechMesh 2026 | Technical Module

Category: Technical Core

Format: Team-Based Web Development Challenge

Duration: 1 Day (Live Competition)

1. Overview

Webathon is a live web development competition in which teams design and build a functional web solution based on a problem statement announced on the event day. Teams will be assessed on functionality, user experience, code quality, creativity, and presentation.

2. Team Composition

Detail Requirement

4. Challenge Requirements

Teams must develop a responsive web solution for the assigned problem statement. The solution must include, at minimum:

  • A home page
  • A core feature workflow representing the main user action
  • Basic data handling, including form input, validation, and display
  • Error handling for invalid input
  • A mobile-friendly and responsive layout

Teams may choose to build:

  • A frontend-only solution using mock or static data
  • A full-stack application, which will be considered a bonus if a working backend is included

All core code must be written during the competition hours.

5. Allowed Technologies

The following technologies are permitted:

  • HTML, CSS, JavaScript, and TypeScript
  • React, Vue, or other frontend frameworks
  • Node.js, Express, Firebase, Supabase, or similar backend tools
  • UI libraries and CSS frameworks
  • Open-source packages
  • AI tools, provided they are used only as assistance and not for full project generation or blind copy-paste

6. Internet and Resource Policy

  • Internet access is allowed for documentation, package installation, and debugging references.
  • Copying complete templates, cloned projects, or full repositories is strictly prohibited.
  • Teams must be able to explain every major part of their code during evaluation.

7. Submission Requirements

Each team must submit the following:

  • Source code as a ZIP file or repository link
  • A README file containing:
  • Project overview
  • Setup instructions
  • Tech stack
  • Team member names
  • Demo evidence, which may include:
  • A live project link, or
  • A recorded walkthrough of 2 to 4 minutes, or
  • An on-site runnable project
  • Optional supporting material:
  • Architecture diagram
  • Known limitations

10. Tie-Break Policy

If two or more teams achieve the same score:

  • The team with higher Functionality marks will rank higher.
  • If still tied, the team with higher UI/UX marks will rank higher.
  • If still tied, judges may conduct a short technical Q&A to decide the final ranking.

11. Code of Conduct

All participants must:

  • Respect organizers, judges, and fellow participants
  • Refrain from harassment, abuse, or disruptive behavior
  • Accept that judges' decisions are final

12. Organizer Notes

  • Organizers may adjust the schedule according to event flow.
  • Any rule changes will be communicated equally to all teams before implementation.

Speed Coding

Task

Participants will be given a coding task on the spot during the competition.

Rules

  • Participants have 2 hours to complete the task.
  • Participants must bring their own laptops.
  • Participants must arrange their own internet connection.
  • Use of AI tools and the internet is allowed.
  • All code must be written during competition time.

Winning Criteria

  • Completion Time: Faster solutions are preferred.
  • Efficiency: Optimized and well-structured code.
  • Uniqueness: Creativity and originality in solving the problem.
  • Interface and Presentation: Quality of user interface and usability of solution.

Speed Wiring

Materials Provided

  • Electronic wires
  • Breadboard
  • Required ICs and basic components

Rules

  • Maximum 3-4 members in one team.
  • Task will be given on the spot.
  • Teams must use only provided components.
  • Participants have 30 minutes to complete the circuit.
  • External materials or components are not allowed.

Winning Criteria

  • Completion Time
  • Workable Circuit: Circuit must function correctly.
  • Logic: Proper understanding and implementation of electronic design.

Speed Mapping

Format

  • Competition is conducted in 1 vs 1 format.

Rules

  • The host will provide a design or mapping task.
  • Each participant must replicate the given design as quickly as possible.
  • Competition is conducted in multiple knockout rounds.
  • Winner of each round advances to next stage.

Winning Criteria

  • The participant who completes the design first and correctly wins the round and advances.

Tech Debate

Format

Competition is conducted in debate format where participants present arguments on technology-related topics.

Rules

  • Debate topics are announced on the spot.
  • Participants are given short preparation time before debate starts.
  • Each participant or team gets fixed time to present arguments.
  • Discussion must remain focused on technology-related aspects.
  • Respectful and professional communication must be maintained.

Winning Criteria

  • Argument Strength: Logical and well-supported arguments.
  • Technical Knowledge: Understanding of the technology topic.
  • Confidence and Communication: Clarity, confidence, and presentation skills.
  • Rebuttal Ability: Effective response to opponent arguments.
